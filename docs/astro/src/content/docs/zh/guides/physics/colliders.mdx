每个物理实体需要在 [RigidBody](./rigidbody/) 旁附加至少一个碰撞体形状。碰撞体定义用于碰撞检测的物理形状。

## 碰撞体类型

| 碰撞体 | 形状 | 适用场景 |
|--------|------|----------|
| [BoxCollider](#boxcollider) | 轴对齐矩形 | 箱子、平台、墙壁 |
| [CircleCollider](#circlecollider) | 圆形 | 球、金币、圆形物体 |
| [CapsuleCollider](#capsulecollider) | 胶囊（矩形两端半圆） | 角色、药丸形物体 |
| [SegmentCollider](./segment-collider/) | 线段 | 薄墙、单向平台 |
| [PolygonCollider](./polygon-collider/) | 凸多边形（最多 8 顶点） | 自定义凸形状 |
| [ChainCollider](./chain-collider/) | 线段链 | 地形、复杂静态边界 |

## 通用属性

所有碰撞体共享以下属性（ChainCollider 不支持 `density` 和 `isSensor`）：

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `density` | number | `1.0` | 质量密度。影响刚体总质量 |
| `friction` | number | `0.3` | 表面摩擦力（0 = 冰面，1 = 橡胶）。ChainCollider 默认值：`0.6` |
| `restitution` | number | `0.0` | 弹性（0 = 无弹性，1 = 完全弹性） |
| `isSensor` | boolean | `false` | 触发区域——检测重叠但不产生物理响应 |
| `categoryBits` | number | `1` | 碰撞过滤类别位 |
| `maskBits` | number | `0xFFFF` | 碰撞过滤掩码位 |
| `enabled` | boolean | `true` | 启用/禁用碰撞体 |

<Aside type="tip">
  将 `isSensor` 设为 `true` 可创建触发区域，触发[传感器事件](./events/)而不阻挡移动。
</Aside>

## BoxCollider

轴对齐矩形碰撞体。

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `halfExtents` | Vec2 | `{x: 0.5, y: 0.5}` | 半宽和半高，以物理单位计 |
| `offset` | Vec2 | `{x: 0, y: 0}` | 相对实体中心的偏移，以物理单位计 |
| `radius` | number | `0.05` | 圆角半径。大于 0 时创建圆角矩形 |

加上所有[通用属性](#通用属性)。

## CircleCollider

圆形碰撞体。

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `radius` | number | `0.5` | 圆半径，以物理单位计 |
| `offset` | Vec2 | `{x: 0, y: 0}` | 相对实体中心的偏移，以物理单位计 |

加上所有[通用属性](#通用属性)。

## CapsuleCollider

胶囊形碰撞体（矩形上下两端为半圆）。

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `radius` | number | `0.25` | 胶囊半径，以物理单位计 |
| `halfHeight` | number | `0.5` | 矩形中段的半高，以物理单位计 |
| `offset` | Vec2 | `{x: 0, y: 0}` | 相对实体中心的偏移，以物理单位计 |

加上所有[通用属性](#通用属性)。

## 碰撞过滤

碰撞层让你控制哪些对象可以互相碰撞。系统使用 16 个命名层和位掩码过滤。

### 工作原理

每个碰撞体有一个 `categoryBits` 字段将其分配到一个层。**Settings → Physics → Collision Matrix** 中的碰撞矩阵决定哪些层之间可以交互。

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `categoryBits` | number | `1` | 碰撞体所属层（位掩码，每层一个位） |

当两个碰撞体的层在碰撞矩阵中被允许交互时，碰撞才会发生。

### 编辑器设置

1. 在 **Settings → Physics → Collision Layers** 中**命名层**（最多 16 层，如 "Player"、"Enemy"、"Ground"、"Projectile"）
2. 在 **Settings → Physics → Collision Matrix** 中**配置矩阵**——勾选需要碰撞的层对
3. 在 Inspector 中**分配层**——每个碰撞体的 `categoryBits` 下拉菜单显示已命名的层

<Aside type="note">
碰撞层完全通过编辑器设置配置。物理插件会自动从项目配置中读取层名称和碰撞矩阵，无需编写代码。
</Aside>
