This guide will help you set up your development environment for building games with ESEngine.

## Prerequisites

Before you begin, ensure you have the following installed:

| Tool | Version | Purpose |
|------|---------|---------|
| CMake | 3.16+ | Build system |
| C++ Compiler | C++17 support | GCC 8+, Clang 7+, MSVC 2019+ |
| Emscripten | 3.1+ | WebAssembly compilation |
| Git | Any | Clone the repository |

## Installing Emscripten

<Tabs>
  <TabItem label="Linux/macOS">
    ```bash
    # Clone emsdk
    git clone https://github.com/emscripten-core/emsdk.git
    cd emsdk

    # Install and activate latest version
    ./emsdk install latest
    ./emsdk activate latest

    # Add to your shell profile
    echo 'source "/path/to/emsdk/emsdk_env.sh"' >> ~/.bashrc
    ```
  </TabItem>
  <TabItem label="Windows">
    ```powershell
    # Clone emsdk
    git clone https://github.com/emscripten-core/emsdk.git
    cd emsdk

    # Install and activate latest version
    .\emsdk.bat install latest
    .\emsdk.bat activate latest

    # Activate in current terminal
    .\emsdk_env.bat
    ```
  </TabItem>
</Tabs>

Verify the installation:

```bash
emcc --version
# emcc (Emscripten gcc/clang-like replacement) 3.x.x
```

## Getting ESEngine

<Steps>

1. **Clone the repository**

   ```bash
   git clone https://github.com/esengine/microes.git
   cd esengine
   ```

2. **Initialize submodules** (if any)

   ```bash
   git submodule update --init --recursive
   ```

3. **Verify the structure**

   ```
   esengine/
   ├── src/esengine/    # Engine source
   ├── include/         # Public headers
   ├── third_party/     # Dependencies (GLM)
   ├── examples/        # Example projects
   └── CMakeLists.txt   # Build configuration
   ```

</Steps>

## Building

### Native Build (for testing)

```bash
cmake -B build -DCMAKE_BUILD_TYPE=Debug
cmake --build build
```

### Web Build

```bash
# Ensure Emscripten is activated
source /path/to/emsdk/emsdk_env.sh  # Linux/macOS
# or: .\emsdk_env.bat                # Windows

emcmake cmake -B build_web -DES_BUILD_WEB=ON
cmake --build build_web
```

### WeChat MiniGame Build

```bash
emcmake cmake -B build_wxgame -DES_BUILD_WXGAME=ON
cmake --build build_wxgame
```

## CMake Options

| Option | Default | Description |
|--------|---------|-------------|
| `ES_BUILD_WEB` | OFF | Build for web browsers |
| `ES_BUILD_WXGAME` | OFF | Build for WeChat MiniGames |
| `ES_BUILD_EXAMPLES` | ON | Build example projects |
| `ES_BUILD_TESTS` | OFF | Build unit tests |

## IDE Setup

### Visual Studio Code

Recommended extensions:
- C/C++ (Microsoft)
- CMake Tools
- clangd (for better intellisense)

`.vscode/settings.json`:
```json
{
  "cmake.configureArgs": ["-DCMAKE_EXPORT_COMPILE_COMMANDS=ON"],
  "C_Cpp.default.configurationProvider": "ms-vscode.cmake-tools"
}
```

### CLion

CLion works out of the box with CMake projects. Open the `esengine` folder and it will auto-detect the CMakeLists.txt.
